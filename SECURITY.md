# Security Policy

## Reporting

Do not open a public issue for a suspected security or privacy vulnerability. Use GitHub's private vulnerability-reporting channel once enabled, or contact the repository owner directly through a private channel.

Include the affected path or object, impact, reproduction steps, and any evidence needed to validate the report. Do not access, retain, or disclose data beyond what is necessary to demonstrate the issue.

## Supported code

Until a production release is declared, only the current `main` branch is supported. Prototype interfaces and data are not production systems.

## Sensitive data prohibition

This public repository must not contain:

- customer or worker information;
- confidential business records;
- attorney-client communications or work product;
- credentials, tokens, certificates, or private keys;
- production configuration or exports;
- non-public source artifacts that cannot lawfully be redistributed.

If sensitive material is committed, stop work, notify the owner privately, revoke affected credentials, remove the material from history, and document the incident.
